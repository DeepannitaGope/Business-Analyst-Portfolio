# Acceptance Criteria
## Overview

This folder contains Acceptance Criteria documentation prepared for the **Claims Management** module of a sample **Insurance Claims Management System**. The documents were created as part of my Business Analyst portfolio to demonstrate how business requirements are translated into clear, testable acceptance criteria for Agile delivery teams.

The documentation follows a structured approach commonly used in enterprise software projects and is intended to showcase requirement analysis, functional documentation, and requirement validation skills.

---

## Project Overview

The Claims Management module covers the complete insurance claim lifecycle, including:

- First Notice of Loss (FNOL)
- Claim Registration
- Supporting Document Upload
- Claim Verification
- Claim Approval / Rejection
- Claim Settlement
- Customer Notifications
- Audit Logging
